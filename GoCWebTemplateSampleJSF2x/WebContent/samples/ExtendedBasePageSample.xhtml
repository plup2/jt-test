GoC Web Template Samples - Extended BasePage

This page provides an example on how to extend the goc.webtemplate.component.jsf.BaseBean class provided by the Web Template. The BaseBean provided by the Web Template has very little functions and therefore applications may wish to extend its functionality. This will be useful should you require to add your own common logic that will be used by your applications' pages.

In this sample, we've created a new class "ExtendedBasePageSample", which extend the BaseBean class from the Web Template and it includes the following:

  • A getter method "getWeather", which returns a string.
  • A getter method "getSessionId", which returns the ID of the user's session
  • Override the "setHeaderTitle" getter method of the WebTemplate. This would set the page title for all pages of the application.

This particular jsf page initializes the "ExtendedBasePage" class in the beaninit section that is made available by the web template, which allows it to have access to the methods of the "ExtendedBasePage" bean but also the methods of the Web Template "BaseBean". In the sample the SessionID and GetWeather are called and the value is displayed on screen.

Level of inheritance:

  • Your pages will inherit from your Extended Base Page
  • Your Extended Base Page will inherit from the goc.webtemplate.component.jsf.BaseBean of the Web Template
  • The goc.webtemplate.component.jsf.BaseBean of the Web Template inherits from goc.webtemplate.component.BaseComponent class

Steps to Extend the Template BasePage

  • Create a new class
  • Inherit from "goc.webtemplate.component.jsf.BaseBean"
  • Include the logic required for your application
import goc.webtemplate.component.jsf.BaseBean;

import javax.enterprise.context.RequestScoped;
import javax.faces.context.FacesContext;
import javax.inject.Named;
import javax.servlet.http.HttpServletRequest;

@Named("extendedbasepagesamplebean")
@RequestScoped
public class ExtendedBasePageSample extends BaseBean {

	@Override
	public void setHeaderTitle() {
		this.headerTitle = "Title set for everpage!";
	}
	
	...
	
	public String getWeather() { return "Sunny"; }
	
	public String getSessionId() {
		HttpServletRequest req = (HttpServletRequest)FacesContext.getCurrentInstance().getExternalContext().getRequest();
		return req.getSession().getId();
	}
	
}

Steps when adding your JSF Pages

  • include the custom bean class in the beaninit section that is made available the web template, always under the name clientbean.
<ui:define name="beaninit">
	<ui:param name="goctemplateclientbean" value="" />		
</ui:define>

Values from the "ExtendedBasePage"

Session ID: #{goctemplateclientbean.sessionId}
Today's Weather: #{goctemplateclientbean.weather}
